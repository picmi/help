# Adding Library Items to workflows

Use the Library panel in the Workflow Builder to add reusable cross-workflow items to a workflow.

Library Items can include:

- Questions
- Information
- Content Blocks—Page Title, Heading, Video, Image
- Contracts

## Add an item at a selected position

::: instructions

1. Go to **Setup** > **Workflows**.
2. Locate the workflow row, click &vellip; (vertical dots) to open the menu.
3. Select **Edit** mode.
4. Select the page or item near where the new Library Item should appear.
5. Open the **Library** panel.
6. Search or filter for the Library Item.
7. Add it **Before** or **After** the selected page or item.
8. Check the placement in the workflow.
9. Save the workflow.

:::

## Add an item with drag and drop

::: instructions

1. Go to **Setup** > **Workflows**.
2. Locate the workflow row, click &vellip; (vertical dots) to open the menu.
3. Select **Edit** mode.
4. Open the **Library** panel.
5. Search or filter for the Library Item.
6. Drag the Library Item into the workflow.
7. Drop it where it should appear.
8. Save the workflow.

:::

Use drag and drop when the placement is easier to choose visually. Use selected-position insertion when you need a
precise before or after placement.

## Locate an already linked Library Item

::: instructions

1. Go to **Setup** > **Workflows**.
2. Locate the workflow row, click &vellip; (vertical dots) to open the menu.
3. Select **Edit** mode.
4. Open the **Library** panel.
5. Search for the Library Item.
6. If the item is already linked, use the linked item :::icon crosshairs-gps::: locate action.
7. Select the located item, click edit :::icon mdi-pencil::: and review it in the Inspector.

:::

## Hide already linked items or show hidden items

Use Library filters to control what appears while you are adding items.

- Hide already linked items when you only want to find items that are not in the workflow.
- Show hidden items when you need to review or restore older Library Items.

::: prompt
A hidden Library Item may still be used in existing workflows. Hiding an item from Library lists does not remove it from
workflows where it is already linked.
:::

## Contracts and opportunity classification

Contract and template search results may be filtered by the workflow's opportunity classification. For example, an
employment workflow should use employment contract templates, and a services workflow should use service agreement
templates.

If the contract you expect is not available, check that the workflow and contract template are for the same opportunity
classification.
