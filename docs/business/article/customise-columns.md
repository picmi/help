# Customise columns

In PICMI, many tables allow you to choose which columns are shown. **Customise Columns** helps you focus on the information you need without changing the underlying records.

::: prompt
Use **Search filters** to narrow which records appear. Use **Customise Columns** to change which details are shown for those records. See [Search filters](search.md).
:::

## A customise columns interaction

Customise columns is opened from :::icon cog-outline::: on a table.

The panel usually includes:

* **Column groups**: related columns grouped into sections such as details, application, contract, or integrations
* **Checkboxes**: select the columns to show
* **Clear**: remove the current selections in the panel
* **All**: select all available columns in the panel
* **Reset**: return to the previous saved state
* **Save**: apply the selected columns to the table

:::: explanation
## What customise columns changes

* **Shows or hides columns** in the current table
* **Does not filter records** from the results
* **Does not change the data** stored in PICMI
* **Can help with quick review** when you need to compare multiple records at once

The available columns will vary depending on the area you are viewing.
::::

:::: instructions
## Open customise columns

1. Go to the page with the table you want to change
2. Click :::icon cog-outline::: **Customise Columns**

The column selection panel will open.
::::

:::: instructions
## Show or hide columns

1. Open :::icon cog-outline::: **Customise Columns**
2. Locate the column group you need
3. Select the checkboxes for the columns to show
4. Clear the checkboxes for the columns to hide
5. Click **Save**

The table will refresh using the selected columns.
::::

:::: instructions
## Select all columns

1. Open :::icon cog-outline::: **Customise Columns**
2. Click **All**
3. Click **Save**

All available columns in that panel will be shown in the table.
::::

:::: instructions
## Clear the selected columns

1. Open :::icon cog-outline::: **Customise Columns**
2. Click **Clear**
3. Click **Save**

The table will update with the cleared selection.

::: prompt
Some areas may always keep a small number of essential columns visible.
:::
::::

:::: instructions
## Reset before saving

1. Open :::icon cog-outline::: **Customise Columns**
2. Make changes to the selections
3. Click **Reset** if you want to return to the previously saved state
4. Click **Save** when ready

Reset is useful if you changed several selections and want to start again from the last saved view.
::::

## Why customise columns is useful

Custom columns are helpful when:

* reviewing many records side by side
* checking a status or field that is hidden by default
* reducing clutter before working through a task
* preparing a table view before selecting records in bulk

::: prompt
If the table looks wrong after changing columns, reopen :::icon cog-outline::: **Customise Columns** and use **All** or **Reset** to restore a more complete view.
:::

## Remembering column settings

PICMI may remember your column choices when you return to the same area.

::: prompt
If another person is following your instructions, they may not see the same columns until they customise their own view.
:::
